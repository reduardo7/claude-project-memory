# Session 2026-05-07

## Context

User reported error in a target project: `Agent type 'memory-digest-daily' not found`. The error occurred when the memory-digest flow tried to invoke `claude-project-memory:memory-digest-daily` as an Agent subagent_type.

Root cause: `skills/memory/context.md` showed `Agent(subagent_type: "claude-project-memory:memory-search", ...)` as an example, which misled Claude into thinking plugin skills can be used as agent types. They cannot — only built-in types (`general-purpose`, `Explore`, `Plan`, etc.) work as `subagent_type`.

## Decisions

- `memory-search` must be invoked via `Skill(skill: "claude-project-memory:memory-search", args: "...")` — runs inline in the current context
- `memory-digest-daily` and `memory-digest-spec` are NOT agent types; their SKILL.md content is read and injected into a `general-purpose` agent prompt
- Removed dead exclusion check from `memory_pre_agent_reminder.py` (the set `{"claude-project-memory:memory-search", ...}` never matched anything since those are not valid agent types)

## Errors and corrections

User showed: "No funciona ésto, arroja error: Agent type 'memory-digest-daily' not found"

Fix applied: corrected all documentation and hook files that used `Agent(subagent_type: "claude-project-memory:...")` syntax.

## Learnings

- Plugin skills are available as: slash commands (`/plugin:skill`) and via Skill tool (`Skill(skill: "plugin:skill")`)
- Plugin skills are NOT available as Agent subagent_type — attempting it fails with "Agent type X not found"
- The previous session response (before this fix) incorrectly stated that plugin skills can be used as `subagent_type`
